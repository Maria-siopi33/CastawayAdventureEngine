# CastawayAdventureEngine
🕹️ Castaway Adventure Engine
Μια επαναχρησιμοποιήσιμη, data-driven μηχανή παραγωγής text-based παιχνιδιών περιπέτειας, αναπτυγμένη εξ ολοκλήρου σε περιβάλλον Java. Η αρχιτεκτονική της μηχανής επιτρέπει τον πλήρη διαχωρισμό της λογικής του κώδικα από το σενάριο του παιχνιδιού, το οποίο φορτώνεται δυναμικά μέσω αρχείων JSON.

🛠️ Οδηγίες Εγκατάστασης & Ρύθμισης (IntelliJ IDEA)
Για να εκτελέσετε το project τοπικά στον υπολογιστή σας, ακολουθήστε τα παρακάτω βήματα για τη σωστή σύνδεση της εξωτερικής βιβλιοθήκης JSON:

Ανοίξτε τον φάκελο του project μέσα στο IntelliJ IDEA.

Πλοηγηθείτε στο μενού: File -> Project Structure -> Libraries.

Πατήστε το εικονίδιο + (New Project Library) και επιλέξτε Java.

Εντοπίστε και επιλέξτε το αρχείο json-20240303.jar το οποίο βρίσκεται αποθηκευμένο μέσα στον φάκελο lib/ του project.

Πατήστε Apply και OK.

⚠️ Προσοχή: Η προσθήκη της βιβλιοθήκης είναι απαραίτητη για τη λειτουργία του WorldBuilder και την ορθή μεταγλώττιση (compile) του κώδικα χωρίς σφάλματα.

🚀 Βασικές Λειτουργίες του Engine
Δυναμική Φόρτωση Κόσμου: Ο WorldBuilder αναλαμβάνει το parsing του αρχείου world.json, κατασκευάζοντας τον χάρτη του παιχνιδιού στο runtime.

Διαχείριση Κατάστασης (State Management): Το GameContext λειτουργεί ως η κεντρική πηγή αλήθειας, αποθηκεύοντας ανά πάσα στιγμή τη θέση του παίκτη, το inventory και την κατάσταση του περιβάλλοντος.

Αντικειμενοστρεφής Σχεδίαση: Υλοποίηση καθαρών αρχών (Separation of Concerns, Dependency Injection) για την αποσύνδεση των εντολών (Commands) από τον κεντρικό ελεγκτή.

📖 Τεκμηρίωση Βασικής Προγραμματιστικής Λογικής
Για την επέκταση της μηχανής με νέες εντολές (π.χ. GoCommand, LookCommand), χρησιμοποιείται η παρακάτω προγραμματιστική διεπαφή (API):

1. Ανάκτηση Θέσης Παίκτη
Η πρόσβαση στο τρέχον δωμάτιο γίνεται μέσω του context:

Java
Room currentRoom = context.getCurrentRoom();
2. Έλεγχος & Διαχείριση Εξόδων
Κάθε δωμάτιο παρέχει τη μέθοδο ανάκτησης εξόδου με βάση την κατεύθυνση:

Java
String nextRoomId = currentRoom.getExit("north");
Αν η μέθοδος επιστρέψει null, η κατεύθυνση είναι μη έγκυρη (δεν υπάρχει έξοδος).

Αν επιστρέψει ένα έγκυρο ID (π.χ. "Jungle"), η μετακίνηση είναι εφικτή.

3. Μετακίνηση Παίκτη
Η ενημέρωση της θέσης του παίκτη στον κόσμο πραγματοποιείται με την ανάθεση του νέου δωματίου στο context:

Java
context.setCurrentRoom(world.get(nextRoomId));
